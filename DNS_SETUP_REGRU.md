# Настройка DNS для sybota.space в reg.ru

## Проблема
Страница не загружается - возможно, проблема в настройке DNS.

## Что нужно проверить в reg.ru:

### 1. Войдите в панель управления reg.ru

1. Откройте https://www.reg.ru
2. Войдите в аккаунт
3. Перейдите в раздел "Домены" → "Мои домены"
4. Найдите домен `sybota.space` и нажмите "Управление"

### 2. Проверьте DNS-серверы

Убедитесь, что используются DNS-серверы reg.ru (или ваши собственные).

### 3. Настройте A-запись

В разделе "DNS-серверы и зона" или "Управление DNS" добавьте/проверьте A-запись:

**A-запись:**
- **Имя/Поддомен:** `@` (или оставьте пустым для основного домена)
- **Тип:** A
- **Значение/IP:** `46.191.230.86`
- **TTL:** 3600 (или автоматически)

**Для www:**
- **Имя/Поддомен:** `www`
- **Тип:** A
- **Значение/IP:** `46.191.230.86`
- **TTL:** 3600

### 4. Проверьте текущие записи

Убедитесь, что нет конфликтующих записей:
- Удалите старые A-записи, которые указывают на другие IP
- Убедитесь, что нет CNAME записей, которые конфликтуют с A-записью

### 5. Сохраните изменения

После настройки DNS изменения могут применяться от нескольких минут до 24 часов (обычно 5-30 минут).

## Проверка DNS после настройки:

```bash
# Проверка A-записи
nslookup sybota.space
# или
dig sybota.space +short

# Должен вернуть: 46.191.230.86
```

## Альтернатива: Использование IP напрямую

Если DNS еще не применился, можно временно использовать IP напрямую:
- http://46.191.230.86

Но для работы Telegram авторизации нужен домен, поэтому DNS обязательно нужно настроить.

## Частые проблемы:

### 1. DNS еще не применился
- Подождите 5-30 минут после изменения
- Очистите DNS кэш: `sudo systemd-resolve --flush-caches` (Linux) или перезапустите роутер

### 2. Неправильный IP в A-записи
- Убедитесь, что указан именно `46.191.230.86`

### 3. Конфликтующие записи
- Убедитесь, что нет других A-записей для `@` или `www`

### 4. Проблема с файрволом
- Убедитесь, что порты 80 и 443 открыты (см. инструкции выше)

## Проверка работы после настройки DNS:

```bash
# С сервера
curl -I http://sybota.space

# Должен вернуть HTTP 200 или 301/302
```

## Скриншоты для reg.ru (примерная навигация):

1. **Домены** → **Мои домены** → **sybota.space** → **Управление**
2. **DNS-серверы** или **Управление DNS**
3. **Добавить запись** → Выбрать тип **A**
4. Заполнить:
   - Имя: `@` (для основного домена) или `www` (для поддомена)
   - Значение: `46.191.230.86`
   - TTL: `3600`
5. **Сохранить**

## Если ничего не помогает:

1. Проверьте, что nginx работает: `sudo systemctl status nginx`
2. Проверьте, что порты открыты: `ss -tulpn | grep :80`
3. Проверьте логи nginx: `sudo tail -f /var/log/nginx/sybota.space.error.log`
4. Попробуйте открыть по IP: http://46.191.230.86 (если работает - проблема точно в DNS)

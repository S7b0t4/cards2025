# Исправление загрузки Telegram Bot Token

## Проблема
Переменная окружения `TELEGRAM_BOT_TOKEN` не загружалась в контейнер backend, хотя она была указана в `backend/.env`.

## Решение

### 1. Добавлен `env_file` в docker-compose.yml

В секции `backend` добавлено:
```yaml
env_file:
  - backend/.env
```

Это позволяет Docker Compose загружать переменные из файла `backend/.env`.

### 2. Переменная добавлена в `environment`

Также добавлено в `environment`:
```yaml
TELEGRAM_BOT_TOKEN: ${TELEGRAM_BOT_TOKEN}
```

Это позволяет переопределить значение через переменную окружения хоста, если нужно.

## Как это работает

1. Docker Compose сначала загружает переменные из `backend/.env` через `env_file`
2. Затем применяет значения из `environment`, которые могут переопределить значения из `env_file`
3. Если переменная `TELEGRAM_BOT_TOKEN` установлена в системе (хосте), она будет использована
4. Иначе будет использовано значение из `backend/.env`

## Проверка

Проверьте, что токен загружен:
```bash
docker-compose exec backend env | grep TELEGRAM_BOT_TOKEN
```

Должно показать токен (не пустое значение).

## После исправления

Теперь авторизация через Telegram должна работать, так как токен бота правильно загружается в контейнер.

# Исправление ошибки "Bot domain invalid"

## Проблема:
Telegram виджет показывает "Bot domain invalid" - это означает, что домен не принят Telegram.

## Решения:

### 1. Проверьте настройку домена в BotFather:

В @BotFather выполните:
```
/setdomain
@S7b0t4bot
cards.sybota.space
```

**Важно:** Telegram может требовать:
- Домен должен быть доступен по HTTPS
- Домен должен иметь валидный SSL сертификат
- Может потребоваться время (до 10-15 минут) для применения изменений

### 2. Проверьте, что домен доступен:

```bash
curl -I https://cards.sybota.space
```

Должен вернуть HTTP 200.

### 3. Проверьте Telegram виджет на странице:

Откройте https://cards.sybota.space/auth и проверьте в консоли браузера (F12):
- Нет ли ошибок загрузки виджета
- Правильно ли указан bot_name в коде

### 4. Временно отключена проверка хеша:

Для отладки проверка хеша временно отключена (`SKIP_TELEGRAM_HASH_CHECK=true`).

После того, как домен заработает, включите обратно:
```bash
cd ~/proj
sed -i 's/SKIP_TELEGRAM_HASH_CHECK=true/SKIP_TELEGRAM_HASH_CHECK=false/' backend/.env
docker-compose restart backend
```

### 5. Альтернатива - используйте тестовый аккаунт:

Пока домен не настроен, можно использовать кнопку "Войти как тестовый аккаунт" на странице авторизации.

## Проверка:

1. Убедитесь, что домен `cards.sybota.space` указан в BotFather
2. Подождите 10-15 минут после настройки
3. Очистите кеш браузера
4. Попробуйте снова

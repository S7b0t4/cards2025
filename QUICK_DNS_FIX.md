# Быстрая настройка DNS в reg.ru для sybota.space

## Шаги в панели reg.ru:

### 1. Войдите в панель управления
- Откройте https://www.reg.ru
- Войдите в аккаунт

### 2. Найдите домен sybota.space
- Перейдите: **Домены** → **Мои домены**
- Найдите `sybota.space` и нажмите **"Управление"** или **"Настроить"**

### 3. Перейдите в настройки DNS
- Найдите раздел **"DNS-серверы"** или **"Управление DNS"** или **"Зона DNS"**
- Нажмите на него

### 4. Добавьте/измените A-запись

**Для основного домена (sybota.space):**
- **Тип записи:** A
- **Имя/Поддомен:** `@` (или оставьте пустым, или введите `sybota.space`)
- **Значение/IP-адрес:** `46.191.230.86`
- **TTL:** `3600` (или оставьте по умолчанию)
- Нажмите **"Добавить"** или **"Сохранить"**

**Для www (www.sybota.space):**
- **Тип записи:** A
- **Имя/Поддомен:** `www`
- **Значение/IP-адрес:** `46.191.230.86`
- **TTL:** `3600`
- Нажмите **"Добавить"** или **"Сохранить"**

### 5. Удалите старые записи (если есть)
- Если есть другие A-записи для `@` или `www` с другими IP - удалите их
- Оставьте только записи с IP `46.191.230.86`

### 6. Сохраните изменения
- Нажмите **"Сохранить"** или **"Применить"**

## Время применения DNS

После сохранения DNS изменения применяются:
- **Минимум:** 5-10 минут
- **Обычно:** 15-30 минут
- **Максимум:** до 24 часов (редко)

## Проверка после настройки

Подождите 10-15 минут, затем проверьте:

```bash
# С вашего компьютера (не с сервера)
ping sybota.space

# Должен показать IP: 46.191.230.86
```

Или откройте в браузере: **http://sybota.space**

## Если не работает сразу

1. **Подождите 15-30 минут** - DNS изменения не применяются мгновенно
2. **Очистите кэш DNS на вашем компьютере:**
   - Windows: `ipconfig /flushdns`
   - Linux: `sudo systemd-resolve --flush-caches` или перезапустите роутер
   - Mac: `sudo dscacheutil -flushcache`
3. **Попробуйте открыть в режиме инкогнито** - чтобы избежать кэша браузера
4. **Проверьте с другого устройства/сети**

## Временное решение

Пока DNS не применился, можно использовать IP напрямую:
- **http://46.191.230.86**

Но для работы Telegram авторизации нужен домен, поэтому DNS обязательно нужно настроить.

## Если ничего не помогает

1. Проверьте, что в reg.ru действительно сохранены изменения
2. Убедитесь, что указан правильный IP: `46.191.230.86`
3. Проверьте, что нет конфликтующих записей
4. Свяжитесь с поддержкой reg.ru, если DNS не применяется более 2 часов

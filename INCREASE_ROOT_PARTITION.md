# Увеличение корневого раздела (/) - ОПАСНАЯ ОПЕРАЦИЯ

## ⚠️ ВНИМАНИЕ: Это может привести к потере данных!

## Текущая ситуация:
- **Диск:** nvme0n1 (931.5GB)
- **Корневой раздел (/):** 50GB (используется 38GB, свободно 9GB)
- **Раздел /home:** 880.5GB (используется 439GB, свободно 384GB)

## Варианты решения:

### Вариант 1: Безопасный - переместить данные (рекомендуется)

Переместить большие директории из `/` в `/home`:

```bash
# Переместить Docker данные (если они не в /home)
# Переместить большие кеши
# Переместить временные файлы
```

### Вариант 2: Увеличить раздел (ОПАСНО!)

**Требуется:**
1. Резервное копирование всех данных
2. Размонтирование /home
3. Изменение размера разделов
4. Возможна потеря данных при ошибке

**Шаги:**
1. Создать резервную копию
2. Загрузиться с Live USB
3. Использовать `gparted` или `resize2fs`
4. Уменьшить /home на нужный размер
5. Увеличить / на освобожденное место

## Рекомендация:

**НЕ увеличивать раздел автоматически!** Это слишком опасно.

Лучше:
1. Переместить большие данные из `/` в `/home`
2. Очистить кеш (освободит ~3-4GB)
3. Удалить неиспользуемые Flatpak (освободит ~5GB)

Итого можно безопасно освободить ~8-9GB без риска потери данных.

# Очистка диска - анализ и рекомендации

## Что занимает место

### 1. Docker (основной потребитель)
- **Образы:** ~10.48GB (можно освободить ~10.15GB)
- **Контейнеры:** ~469.8MB
- **Volumes:** ~5.497GB (можно освободить ~4.594GB)

### 2. Рекомендации по очистке

#### Docker очистка (освободит ~15GB):

```bash
# Удалить неиспользуемые образы, контейнеры, volumes и сети
docker system prune -a --volumes -f

# Удалить только неиспользуемые образы
docker image prune -a -f

# Удалить только неиспользуемые volumes
docker volume prune -f

# Удалить только остановленные контейнеры
docker container prune -f
```

#### Очистка кеша пакетов (Arch Linux):

```bash
# Очистить кеш pacman (может освободить несколько GB)
sudo pacman -Sc

# Удалить неиспользуемые пакеты
sudo pacman -Rns $(pacman -Qtdq)
```

#### Очистка логов:

```bash
# Очистить старые логи journalctl
sudo journalctl --vacuum-time=7d

# Или ограничить размер логов до 100MB
sudo journalctl --vacuum-size=100M
```

#### Очистка кеша пользователя:

```bash
# Очистить кеш npm (если используется)
npm cache clean --force

# Очистить кеш pip (если используется)
pip cache purge
```

## Безопасная очистка (рекомендуется)

Выполните по порядку:

```bash
# 1. Очистить Docker (освободит больше всего места)
docker system prune -a --volumes -f

# 2. Очистить кеш pacman
sudo pacman -Sc

# 3. Очистить старые логи
sudo journalctl --vacuum-time=7d

# 4. Проверить результат
df -h /
```

## Что НЕ удалять

- **Активные Docker контейнеры и образы** (proj-frontend, proj-backend, postgres_db)
- **Важные данные в ~/proj**
- **Системные файлы**

## Мониторинг

Проверяйте использование диска регулярно:
```bash
df -h /
docker system df
```
